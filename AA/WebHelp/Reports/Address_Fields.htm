Address Fields

Mailing labels require accurate and well formatted address data; this is also true, to a lesser extent, for contact lists with address data.

In its original form, some of the legacy data for alumni had the whole address in one field; therefore when the legacy data was imported and stored in the Alumni Archive database, the whole postal address was stored in the Street Address.

  • So this Street Address field often includes data that would otherwise be stored in the Business Name, Street Address, City, State / Province, Postal Code, and Country fields.

  • For many records, the data includes the needed carriage returns and can be used to create proper mailing labels, but for many it does not and therefore cannot.

  • And some records lack part or all of the needed address data.

Therefore, when the data is displayed in the Excel spreadsheet, the user will have to examine it. Records missing necessary address data can be deleted from the worksheet before merging and printing.

  • Users are encouraged to make corrections and additions to the data in the database, NOT on the spreadsheet.

  • Corrections should include cutting and pasting the other data (except the actual street address) into the other fields, as appropriate, and deleting everything except the actual Street Address data from that field.

Users can sort the alumni list in the spreadsheet, by clicking on columns and using the sort buttons on the toolbar. This will allow you, for example, to separate those records where the address is still in the one field, but is usable, from any newer or corrected records which must use all of the address fields to produce a usable label.

  • One set of records can be cut and pasted into a separate Excel file, and the two groups printed separately, using different sets of fields.

  • For records where the address is still in one field, users will need to delete all superfluous address fields before merging and printing.
